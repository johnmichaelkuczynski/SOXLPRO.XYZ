Supplement to the Custom Strategy Builder: Expose App-Generated Signals as Indicators.
Extend the Custom Strategy Builder's indicator library to include signals generated by the app's own analytical functions. The user should be able to build strategies that reference these signals just like any other indicator.
Add these to the indicator dropdown:

Vol Surface Signal (Calls) — returns one of BUY, SELL, NEUTRAL per day, based on the BUY/SELL candidate logic in Function 2b. Operators: equals BUY, equals SELL, equals NEUTRAL.
Vol Surface Signal (Puts) — same, for puts.
Vol Regime Label (Function 2c) — returns CHEAP, MID-RANGE, EXPENSIVE. Operators: equals.
SOXL-QQQ Dislocation Z-Score (Function 3) — returns numeric z-score. Operators: >, <.
Probability Engine Output (Function 1c) — returns P(move ≥ M% in H days). Operators: >, <.
Period Analysis Percentile (Function 1b) — returns current return's percentile within the historical distribution. Operators: >, <.

Example strategies the user must be able to build:

"Buy SOXL when Vol Surface Signal (Calls) = BUY, sell when Vol Surface Signal (Calls) = SELL"
"Buy SOXL when Vol Regime = CHEAP AND Dislocation Z-Score < -2, sell when Dislocation Z-Score > 0"
"Buy SOXL when Probability Engine output for +10% in 30d > 0.65"

Critical implementation requirement — no lookahead bias:
Each signal must be computed using only data available on or before the signal date. If the Vol Surface signal for 2024-06-15 uses any option chain data from after 2024-06-15, the backtest is invalid. Every app-generated signal function must be refactored to accept a point_in_time parameter and return only what could have been known at that moment. Label this explicitly in the code as compute_signal_pit(date).
Backtest limitations for options-dependent signals:
The Vol Surface signals (items 1 and 2 above) require historical options chain data. Polygon gives us 4 years. Any custom strategy using these signals must auto-restrict the backtest date range to the Polygon window (2022–present) and display a warning: "Strategy uses Vol Surface signals; backtest limited to 2022–present due to options data availability."
Signals 3–6 use equity data only and backtest against the full 16-year EODHD window.
Save/Load integration:
Custom strategies using app-generated signals should save and load correctly, with the signal definitions versioned. If the signal logic changes later (e.g., you tune the vol surface fit), flag loaded strategies with "This strategy was built against an earlier version of the Vol Surface signal; results may differ."
